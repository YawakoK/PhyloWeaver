# AGENTS.md

このファイルは、このリポジトリで作業するエージェント向けのプロジェクトガイドです。

## プロジェクト概要

- PhyloWeaver は、ブラウザ上で系統樹を読み込み、編集し、SVG/PNG/PDF/NEWICK などへ出力する React + TypeScript アプリです。
- 入力データはブラウザ内で処理されます。NEWICK や系統樹データを外部サーバーへ送信する実装は追加しないでください。
- 主要な編集対象は `src/TreeEditor.tsx` です。NEWICK の parse/serialize、D3 によるレイアウト、選択状態、履歴、検索、エクスポート処理が同じコンポーネント内にまとまっています。

## 開発コマンド

- 依存関係のインストール: `npm install`
- 開発サーバー: `npm run dev`
- 本番ビルド確認: `npm run build`
- Lint: `npm run lint`
- ビルド成果物の確認: `npm run preview`

変更後は、可能な限り `npm run lint` と `npm run build` を実行してください。UI や SVG エクスポートを変えた場合はブラウザでも動作確認してください。

## 実装メモ

- フレームワークは React 19、ビルドは Vite/Rolldown、描画は D3、スタイルは Tailwind CSS です。
- UI 文言は `src/TreeEditor.tsx` の `UI_TEXT` に英語と日本語で定義されています。ユーザー向け文言を追加・変更するときは、原則として `en` と `jp` の両方を更新してください。
- 系統樹データは `TreeNode` 型を中心に扱われます。`__id`、`__edgeColor`、`__edgeWidth`、`__nodeSize`、`__collapsed` などの内部フィールドは描画・選択・編集状態に使われるため、コピーやシリアライズ時の扱いに注意してください。
- ツリーの更新は `commitTree` / `mutateTree` の既存パターンに合わせ、履歴、ズーム、選択状態を壊さないようにしてください。
- NEWICK の読み書きは `parseNewick` と `toNewick` が担当します。フォーマット対応を広げる場合は、既存の枝長、内部ノードラベル、サポート値の扱いを確認してください。
- フィログラムとクラドグラムでは枝長の扱いが異なります。レイアウト関連の変更は `layout`、`xScaleWidth`、`__cladoOffset`、`totalLength` の関係を確認してください。
- SVG/PNG/PDF 出力は、表示中の SVG を加工して `canvg`、`jspdf`、`svg2pdf.js` へ渡します。SVG 要素の属性や構造を変更した場合は、画面表示だけでなく各エクスポートも確認してください。
- アイコンなどの静的アセットは `src/assets/icons/` にあります。既存の SVG import パターンに合わせて追加してください。

## 作業方針

- 既存の単一コンポーネント構造を尊重し、不要な大規模分割やリファクタリングは避けてください。
- 変更は目的に対して小さく保ち、ツリー編集、描画、エクスポート、スタイルのどこに影響するかを確認してください。
- 型エラーを避けるため、`unknown` や内部フィールドを扱うときは明示的に型を絞ってください。
- CSS は主に Tailwind クラスで書かれています。グローバル CSS を変える場合は、アプリ全体への影響を確認してください。
- `package-lock.json` は npm 依存関係を変更したときだけ更新してください。
